# Product

## Register

product

## Users

People who copy rich content from browsers, editors, Microsoft Office, or terminal tools and need clean Markdown quickly. They are usually in a writing, documentation, support, or developer workflow and want the converter to stay out of the way.

## Product Purpose

Paste to Markdown converts clipboard HTML, RTF, and plain text into Markdown locally in the browser. Success means users can paste, inspect or edit the result, preview the rendered Markdown, and copy it back without sending content to a server.

## Brand Personality

Calm, precise, practical. The interface should feel modern and trustworthy, with enough polish to feel current but no decorative weight that slows down the conversion task.

## Anti-references

Avoid old Bootstrap demo pages, busy instruction-heavy layouts, decorative gradients, generic SaaS card grids, and theme controls that look separate from the working surface.

## Design Principles

- Make the paste-to-result workflow the primary object on the page.
- Keep instructions short, visible, and subordinate to the working area.
- Use restrained color for actions, focus, and selected state only.
- Preserve local-only trust with clear utility copy.
- Support light, dark, and system theme choices without layout shifts.

## Accessibility & Inclusion

Target readable contrast in both light and dark modes, visible keyboard focus, semantic controls, and reduced-motion fallbacks. The interface should work with system color preference by default while allowing an explicit user override.
